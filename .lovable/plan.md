## Font (verified)
The current portfolio headings ("Portfolio", "Tech & Apps") use the `font-heading` class. `tailwind.config.ts` maps `fontFamily.heading вҶ’ "DM Serif Display"`, and `index.css` sets `--font-heading: 'DM Serif Display'`. So `font-heading` **is** the portfolio's actual typeface вҖ” I'll reuse `font-heading` for the section heading and all category headers (same DM Serif Display, no substitution).

- Free-standing vertical phone mockups on the clean section background вҖ” no large bordered container boxes. Generous spacing between tiles and categories.
- Subtitle: plain, understated, on-brand copy (no "scroll-stopping"/UGC buzzwords) вҖ” easy to edit.

## Grid вҖ” variable tile counts
- Single responsive grid per category: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` with gap. Because it's a true CSS grid driven by `category.tiles.map(...)`, any count (3, 6, 7вҖҰ) wraps naturally into full rows вҖ” no hardcoded single row of 3.
- Consistent column tracks mean tiles always align; a leftover tile lands left-aligned in a new row rather than breaking the layout.

## Category headers
Serif category name (`font-heading`) followed by a thin horizontal rule extending across the remaining width вҖ” implemented as the name plus a `flex-1` hairline (`border-t` / `bg-border`), consistent with the site's design language. On-brand tokens only (`text-foreground`, `text-primary`, `bg-border`), no hardcoded colors.

## Jump-nav
- A quiet inline row of category names near the top, generated by mapping over the same `categories` array (always in sync).
- Each is an anchor `<a href="#${category.id}">` that smooth-scrolls to the matching category wrapper (`id={category.id}`), with `scroll-mt-24` on each category so headers aren't hidden under the fixed navbar. Smooth scroll via `scroll-smooth` on `html` (already standard) or `scroll-behavior`.
- Styled subtly: small muted text, separated by thin dividers/dots, hover вҶ’ primary. Not a heavy menu.

## Tile design
Each tile is a free-standing vertical phone mockup:
- Portrait `aspect-[9/16]`, rounded corners, soft shadow (`shadow-card` вҶ’ `shadow-elevated` on hover), subtle bezel feel, gentle hover lift.
- Centred play affordance: lucide `Play` icon inside a soft circle overlay.
- Small caption space BELOW the tile for an optional short label (content type). No loud brand headline per tile вҖ” brand context stays quiet (small caption / hover only).
- Placeholder thumbnail background for now.

## Easy swap to real videos later
Typed data array at the top of the file:

```ts
type Tile = { label?: string; thumbnail?: string; videoUrl?: string };
type Category = { id: string; name: string; tiles: Tile[] };
```

- Every tile renders from this data. When `thumbnail` exists it's shown as the tile background; when `videoUrl` exists the tile becomes a link (`<a>` opening the video) вҖ” otherwise it renders the placeholder play state.
- Going live later = fill `thumbnail`/`videoUrl`/`label` per tile. Adding/removing tiles or whole categories = just edit the array; the grid and jump-nav both update automatically.
- Categories seeded: `Product` (lead, top), `Accommodation & Travel`, `Lifestyle & Experience`, each with a few placeholder tiles.
